Operating manual. Only non-discoverable, behavior-changing rules. Run `glob`/`ls` for layout; do not maintain a tree here. ## What this is

AI payment router + agentic orchestration. Multi-provider routing (cost/quality/speed/balanced), x402 settlement (EIP-3009 gasless USDC on Base + Tempo TIP-20), A2A + AP2 over JSON-RPC 2.0, Gemini intelligence, ERC-8004 identity/reputation.

## How to work here

- **Before coding:** check `MEMORY.md` for prior decisions, then read the relevant source-of-truth file from the table below. Do not infer from this doc alone.
- **Trust the type system.** With `strict + noUncheckedIndexedAccess + strictNullChecks`, if it compiles and lint passes, ship. Don't add runtime guards for impossible states.
- **Tests:** run `npm run lint && npm run test:run` before pushing to `main` (ESLint is bypassed in prod builds, so local is the only gate). Skip `test:build` unless you touched routing, middleware, or top-level imports.
- **Migrations:** linear scripts in `scripts/migrations/`, applied in order, schema mirror in `lib/db/schema-v2.sql`. Never edit a shipped migration; add a new one. Order in `DEPLOYMENT.md` §1.
- **Confirm before:** destructive SQL, force-push, force-pushing to a deployed branch, editing live contracts, touching `processed_tx_hashes` / `replay_protection` / `ap2_mandates` (financial integrity).
- **Don't touch:** `cloudflare-facilitator/` (separate project), `contracts/` (Hardhat, tsconfig-excluded), `public/llms-full.txt`, `public/downloads/p402.zip` (generated by `scripts/build-skill.sh`).

- `ApiErrorCode` is a strict union: add the code to `lib/errors.ts` before throwing it.

## Critical gotchas

1. **Redis degrades gracefully; security guards do not.** Code runs with no `REDIS_URL` (`lib/redis.ts`). `BillingGuard` and equivalents must fail-close and propagate, never fall back.
2. **Auth split.** `/api/v2/*` (middleware) needs `x-p402-session`. `/dashboard/*` uses cookie auth via `app/dashboard/layout.tsx`. Never add `/dashboard` to `PROTECTED_API_PATHS`.
3. **Stripe webhook.** Next.js 15 auto-parses JSON, breaking signatures. Use `await req.text()` + `export const dynamic = 'force-dynamic'`.
4. **A2A billing-cap errors.** Never throw HTTP 402 inside A2A. Map via `lib/a2a-errors.ts` to JSON-RPC `-32000`.
5. **API keys.** `p402_live_...` from `crypto.randomBytes`, returned once by the Server Action; only SHA-256 stored.
6. **Idempotency.** Billing events + audit findings use `INSERT ... ON CONFLICT`.
7. **`@coinbase/cdp-sdk` is dynamic-import only.** Top-level import pulls `@solana/kit` (ESM, broken named exports) and crashes the build. Use `await getCdpClientAsync()` from `lib/cdp-client.ts`. Tests mock `@/lib/cdp-client`, not the raw SDK; export `isCdpEnabled`, `getCdpClientAsync`, `getCdpClient`, `_resetCdpClient`.
8. **CDP policy API:** `cdp.policies.createPolicy(...)`, not `cdp.createPolicy(...)`. Body: `{ policy: { scope: 'account', rules: [{ action: 'reject', operation: 'signEvmTransaction', criteria: [...] }] } }`.

### Tempo mainnet (chain 4217)

9. **`0xef` is valid bytecode.** TIP-20 stablecoins return single-byte `0xef` (EIP-7702 designator, Tempo system-contract convention). `code.length > 2` correctly treats deployed. Probe: `tempoMainnetProbe()` in `lib/facilitator-adapters/tempo.ts`.
10. **No `msg.value`.** No native gas token; gas paid in TIP-20 via FeeAMM. Never set non-zero `value`. Use standard ERC-20 `transfer(to, amount)`.
11. **RPC:** `http(process.env.TEMPO_RPC_URL ?? 'https://rpc.tempo.xyz')`. Use `chain: tempo` for metadata only.
12. **Currency is DB-driven.** `auth_config.stablecoin.address` (JSONB on `facilitators`) per row. Canonical list in `lib/constants/tempo.ts`. Drift caught by `lib/__tests__/tempo-currency-sync.test.ts` (opt-in: `TEMPO_SYNC_TEST=true`).
13. **Bigint-clean settlement, 1-raw-unit floor.** `handleOnchainSettle()` passes `BigInt(maxAmountRequired)` straight to `verifyTransaction`. `exact`/`receipt` use viem `formatUnits`/`parseUnits`. Regression: `lib/__tests__/{amount-precision,onchain-settle-precision}.test.ts`.
14. **`handleOnchainSettle` needs a UUID `tenantId`.** `processed_tx_hashes.tenant_id` is `UUID NOT NULL`. Onchain has no session → pass system tenant `00000000-0000-0000-0000-000000000001`. Non-UUID = Postgres 22P02 + 500. Regression: `lib/__tests__/onchain-settle-uuid.test.ts`.

## Protocol facts (non-discoverable)

- **x402 schemes:** `exact` (EIP-3009 TransferWithAuthorization, facilitator pays gas), `onchain` (client submits, facilitator verifies), `receipt` (reuse prior). Min $0.01 USDC on Base. `value` must equal `maxAmountRequired`. Nonce replay-protected (`REPLAY_DETECTED`). EIP-712 in `lib/x402/eip3009.ts`.
- **USDC subscriptions (EIP-2612, separate from x402):** month 1 `executeFirstSubscriptionCharge()` (Server Action `lib/actions/billing-finalize.ts`) sets permit allowance; month 2+ `executeRecurringCharge()` (cron `app/api/internal/cron/billing/reconcile/route.ts`) draws it. Both in `lib/billing/providers/onchain.ts`.
- **AP2 errors:** `MANDATE_NOT_FOUND | _INACTIVE | _EXPIRED | _BUDGET_EXCEEDED | _CATEGORY_DENIED | _SIGNATURE_INVALID`. Validation order: exists+active → not expired → budget → category → signature.
- **A2A x402 extension URI:** `tag:x402.org,2025:x402-payment`.
- **mppx on `/api/v2/chat/completions`** (`USE_MPP_METHOD=true`): `Authorization: Payment <b64url JSON>` → mppx/tempo charge, no API key. Tenant API key → existing billing guard. Neither → 402 with `WWW-Authenticate: Payment` + `X-PAYMENT-REQUIRED`. x402 `X-PAYMENT*` headers guaranteed through May 2027. `Mppx` is in `mppx/nextjs` (not `mppx`); `tempo()` returns `[charge, session]`; amount is human-readable decimal (`'0.001'` = 1000 raw). `store.update()` is deliberately non-atomic — blockchain nonce is the real replay guard (see `lib/mpp/store.ts`).

## CDP <-> AP2 <-> ERC-8004 wiring

Two integration points.

**Session create** (`app/api/v2/sessions/route.ts`), when `wallet_source === 'cdp'` + `agent_id`:
- Auto-issues AP2 mandate (`ap2_mandates`, `type:'payment'`, `user_did:did:p402:tenant:{tenantId}`, `agent_did:did:p402:agent:{agent_id}`, `max_amount_usd = budget_usd`, `valid_until = expiresAt`). Mandate id → `ap2_mandate_id` in `policies` JSONB on `agent_sessions`.
- If `ERC8004_ENABLE_VALIDATION=true` and `agent_id` numeric, fires `setAgentWalletOnChain()` fire-and-forget. Never blocks.

**Auto-pay** (`app/api/v1/router/auto-pay/route.ts`, keyed on `session_token` not `id`): pre-settle, if `ap2_mandate_id` in `session.policies`, call `AP2PolicyEngine.verifyMandate()` (403 on fail); no mandate → skip (backwards compatible). Post-settle non-blocking: `recordUsage()`, `budget_spent_usd` increment, `queueFeedback()`. E2E: `tests/e2e/cdp-ap2-erc8004-wiring.spec.ts`.

## Testing rules

- No DB mocking in unit tests; use fixtures.
- `requireTenantAccess(req)` routes rely on the global mock in `__tests__/setup.ts` (avoids `headers()`/`cookies()` scope errors).
- `npm run test:build` runs a real production build (5m timeout, excluded from default vitest, included in `test:all`).
- Playwright: no extensions, so set NextAuth cookie directly or inject mock provider via `page.addInitScript()` (`tests/e2e/helpers/web3-mock.ts` overrides `eth_signTypedData_v4`). Mock SSE with `route.fulfill`. Mock LLM with `page.route()` except `error-resilience.spec.ts`. Always isolated test DB.
